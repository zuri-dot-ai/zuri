`zuri-stock/` folder of the `dzuvmw4l` Cloudinary account, grouped by
archetype then slot folder. Each entry is the exact `public_id` (== Cloudinary
`storage_path`, no file extension) to use when building the Cloudinary URL:

```
https://res.cloudinary.com/dzuvmw4l/image/upload/<transform>/<public_id>
```

**Append here** whenever new images are uploaded, keeping the same grouping —
this file is the single source of truth Cursor should consult when wiring
real images into any template (V2 samples now, and all 80 templates later).

Note: a small number of entries have a `(1)` suffix (duplicate-upload
artifacts from Cloudinary auto-renaming on re-upload). Prefer the
non-suffixed version when both exist; avoid `(1)` variants in template code
since the literal parentheses/space need URL-encoding.
