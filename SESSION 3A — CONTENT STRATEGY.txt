CREDIT CONSERVATION CHEAT SHEET
Expensive (use sparingly):
  - Multiple back-and-forth messages = N requests, use ONE complete prompt
  - @codebase = scans entire repo, expensive
  - o3-mini / reasoning models = burn fast

Free / cheap:
  - cursor-small = free for inline autocomplete
  - Reading files with @file = minimal cost
  - Composer mode = same cost as chat but produces actual changes

Rules:
  1. Write the full prompt before hitting Enter — no "also do X" follow-ups
  2. Paste the FULL error message into the same thread, not a new one
  3. Use @docs/filename.md not @codebase
  4. One session per domain — don't mix website builder and analytics in one session
  5. When a session succeeds → commit → start fresh thread for the next session
